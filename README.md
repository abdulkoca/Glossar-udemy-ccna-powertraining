# Glossar-udemy-ccna-powertraining


TCP/IP
Transmission Control Protocol/Internet Protocol: Ein grundlegendes Protokoll für den Datenaustausch über das Internet; TCP sorgt für die zuverlässige Lieferung, IP kümmert sich um die Adressierung.
Beispiel: TCP stellt sicher, dass eine E-Mail korrekt und vollständig ankommt.

CLI
Command Line Interface: Eine Text-basierte Möglichkeit, mit Computern zu interagieren, oft verwendet, um Netzwerkgeräte zu konfigurieren.
Beispiel: Konfiguration eines Routers über eine Befehlszeile.

Catalyst
Eine Serie von Netzwerk-Switches von Cisco, bekannt für ihre Leistungsfähigkeit und Funktionalität im Unternehmensbereich.

ISO OSI-Modell
Open Systems Interconnection: Ein Referenzmodell mit sieben Schichten zur Standardisierung der Netzwerkkommunikation.
Beispiel: Die physische Schicht kümmert sich um Kabel und Hardware, die Anwendungsschicht ermöglicht Programme wie Webbrowser.

CSMA/CD
Carrier Sense Multiple Access with Collision Detection: Ein Protokoll zur Vermeidung von Datenkollisionen in Netzwerken.
Beispiel: Ethernet-Netzwerk, das sendet, wenn die Leitung frei ist und auf Kollisionen reagiert.

Bridge
Ein Gerät oder Software, das es ermöglicht, mehrere Netzwerksegmente miteinander zu verbinden und den Datenverkehr zwischen ihnen zu filtern.

Ethernet
Ein Standard für kabelgebundene Netzwerkverbindungen, weit verbreitet für lokale Netzwerke (LANs).
Beispiel: Ein Computer mit Ethernet-Kabel verbundenem Internet-Netzwerk.

ARP
Address Resolution Protocol: Wandelt IP-Adressen in physische MAC-Adressen um.
Beispiel: Wenn ein Computer eine IP-Adresse hat und die physische Adresse des Geräts in einem Netzwerk herausfinden muss.

IPv4
Die vierte Version des Internet Protocol, das IP-Adressen in einem 32-Bit-Format vergibt und ermöglicht die Kommunikation im Netz.

Subnetz
Ein kleineres Netzwerksegment innerhalb eines größeren Netzwerks, welches die Netzwerkleistung und -organisation verbessert.
Beispiel: Aufteilung des Unternehmensnetzwerks in kleinere Untergruppen.

CIDR/VLSM
Classless Inter-Domain Routing/Variable Length Subnet Masking: Techniken zur effizienteren Nutzung von IP-Adressen durch flexible Subnetzmasken.
Beispiel: Eine Firma nutzt CIDR, um IP-Adressen besser auszuschöpfen.

SSH
Secure Shell: Ein Protokoll zur sicheren Fernsteuerung und Verwaltung von Geräten über ungeschützte Netzwerke.
Beispiel: Fernzugriff für sicheren Serverzugriff.

Telnet
Ein älteres Netzwerkprotokoll zur Fernsteuerung von Geräten, unsicherer als SSH.

CDP/LLDP
Cisco Discovery Protocol/Link Layer Discovery Protocol: Cisco-exklusiv/Standardprotokoll zur Geräteerkennung im Netzwerk.
Beispiel: Sichtbare Geräte im Netzwerk finden.

Routing
Der Prozess, Datenpakete basierend auf ihren IP-Adressen von einem Netzwerk zum nächsten zu leiten.

ICMP
Internet Control Message Protocol: Ein Netzwerkprotokoll zur Fehlerdiagnose und Kontrolle von IP-Datenpaketen.
Beispiel: Wird von Tools wie Ping verwendet.

Traceroute
Ein Programm zur Messung und Anzeige der Route von Datenpaketen im Netzwerk.
Beispiel: Verfolgen, welche Serverdaten passieren, um schließlich ein Ziel zu erreichen.

VLAN
Virtual Local Area Network: Ermöglicht logische Gruppierung von Geräten, unabhängig von deren physischer Standort.
Beispiel: Segregation von Abteilungen innerhalb eines Firmennetzwerks.

VTP
VLAN Trunking Protocol: Ermöglicht Verwaltung und Propagierung von VLAN-Informationen in einem Netzwerk.
Inter-VLAN-Routing
Routen von Datenverkehr zwischen verschiedenen VLANs, typischerweise durch einen Router oder Layer-3-Switch realisiert.

Link-State
Ein dynamisches Routing-Protokoll, das Netzwerktopologie für effizientere Routing-Entscheidungen verwendet.

OSPF
Open Shortest Path First: Ein Link-State-Routing-Protokoll, das dazu dient, die effizientesten Netzwerkrouten zu finden.

DR/BDR
Designated Router/Backup Designated Router: In OSPF verwenden Netzwerke DR und BDR zur Optimierung des Netzwerkverkehrs.

STP
Spanning Tree Protocol: Ein Protokoll zur Vermeidung von Netzwerk-Loops in Ethernet-Netzwerken.

PVST, PortFast, BPDU
Per-VLAN Spanning Tree/Bridge Protocol Data Units: Varianten und Schutzmechanismen von STP zur Optimierung und Sicherheit der Netzwerkstruktur.

LACP
Link Aggregation Control Protocol: Ermöglicht Bündelung von Netzwerkports zur Verbesserung der Bandbreite.

EtherChannel
Eine Cisco-Technologie zur Bündelung von mehreren Switch-Ports zur Erhöhung der Bandbreite und Redundanz.

FHRP
First Hop Redundancy Protocol: Protokoll zur Sicherstellung der Netzwerkverfügbarkeit durch Redundanz.

HSRP, VRRP, GLBP
Hot Standby Router Protocol/Virtual Router Redundancy Protocol/Gateway Load Balancing Protocol: Protokolle zur Gewährung von Ausfallsicherheit und Lastverteilung beim Routing.

ACK (Acknowledgement)
Ein Bestätigungspaket, das anzeigt, dass ein Datenpaket erhalten wurde.
Beispiel: Empfangsbestätigung bei TCP-Datenverkehr.

UDP
User Datagram Protocol: Ein schnelles, aber unsicheres Netzwerkprotokoll, das keine Fehlersicherung bietet.
Beispiel: Wird bei Streaming von Videos verwendet.

DHCP
Dynamic Host Configuration Protocol: Automatische Zuweisung von IP-Adressen an Geräte im Netzwerk.

DNS
Domain Name System: Übersetzt Domainnamen in IP-Adressen.
Beispiel: www.beispiel.com wird zu einer numerischen IP-Adresse.

NTP
Network Time Protocol: Synchronisiert die Uhrzeit über Geräte im Netzwerk.
Beispiel: Synchronisierung der Zeit zwischen Servern.

SNMP
Simple Network Management Protocol: Verwalten und Überwachen von Netzwerkgeräten und ihrer Leistung.

SSL/TLS
Secure Sockets Layer/Transport Layer Security: Protokolle zur Sicherung der Kommunikation über das Internet.
Beispiel: Sichere Übermittlung von Kreditkarteninformationen über Webseiten.

ACL
Access Control List: Regeln zur Kontrolle des Zugriffs auf und des Flusses von Netzwerkverkehr.
Beispiel: Gewährung oder Verweigerung von Daten für bestimmte Geräte.

Wildcard
Eine Methode zur Beschreibung von IP-Adressen-Ranges in Netzwerkkonfigurationen, oft in Verbindung mit ACLs verwendet.

NAT
Network Address Translation: Übersetzt private IP-Adressen in eine öffentliche IP-Adresse für die Internetkommunikation.
Beispiel: Heimrouter verwendet NAT, um die Kommunikation von mehreren Geräten mit dem Internet zu ermöglichen.

WAN
Wide Area Network: Verbindet Geräte über große geografische Entfernungen.
Beispiel: Das Internet ist das größte WAN.

VPN
Virtual Private Network: Eine sichere Verbindung über das Internet-Angebot zur Verschlüsselung und Schutz von Daten.
Beispiel: Zugang zu Firmennetzwerk von externem Standort.

IPsec
Internet Protocol Security: Protokolle zur Sicherstellung des geschützten Austauschs von Internetprotokoll-Paketen.

QoS
Quality of Service: Erlaubt der Netzwerkverwaltung priorisierte oder begrenzte Datenverkehrskategorien.
Unicast-Adresse
Eine einzelne Zieladresse für den Versand individueller Pakete an einen spezifischen Empfänger.

IDS/IPS
Intrusion Detection System/Intrusion Prevention System: System zur Erkennung und Verhinderung von Eindringen und Angriffen im Netzwerk.

ARP
Address Resolution Protocol: Bereits erklärt, aber oft doppelt genannt in Netzwerkkontexten.
Durch diese kurze Erklärungstour sollten Anfänger einen grundlegenden Überblick über wichtige Netzwerk-Konzepte und deren Anwendung erhalten.
